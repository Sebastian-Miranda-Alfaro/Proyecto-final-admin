El enfoque del proyecto es interesante desde el punto de vista temático, pero limita su potencial al quedarse en HTML y CSS. Si la investigación es realmente profunda, sería ideal complementarla con herramientas que permitan una experiencia más interactiva, como JavaScript para visualizaciones de procesos, diagramas dinámicos de flujo o una base de datos para registrar y consultar buenas prácticas de ITSM. Aun así, como punto de partida, puede ser una buena introducción a la integración entre teoría de TI y desarrollo web, siempre y cuando se tenga la intención de escalar el proyecto más adelante.
Aunque HTML y CSS no son suficientes para representar la complejidad operativa de marcos como ITIL o COBIT, el proyecto puede tener gran valor si se orienta como un repositorio bien organizado y visualmente claro de información relevante. Para un profesional en TI, la habilidad de comunicar conceptos técnicos de forma sencilla es fundamental, especialmente cuando se habla con directivos o clientes. Un sitio web bien diseñado, aunque básico, puede actuar como un excelente apoyo visual para capacitaciones, presentaciones o documentación interna sobre la gobernanza de TI y la gestión de servicios.
El desarrollo de este proyecto en HTML y CSS, acompañado de una investigación rigurosa sobre COBIT, ITIL, ITSM y gobernanza de TI, representa una excelente oportunidad para que los estudiantes integren conocimientos teóricos con habilidades técnicas básicas. A pesar de que HTML y CSS no son herramientas complejas, el valor educativo radica en cómo el contenido se estructura, presenta y conecta con las metodologías estudiadas. Esta combinación permite que el alumno no solo memorice conceptos, sino que los interprete, los aplique y los comunique claramente, lo cual es esencial para su formación profesional.
